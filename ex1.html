Exercice 1 — Sélecteurs et spécificité

1.1 — Sélecteurs CSS

HTML de référence :

<nav class="main-nav">
  <ul>
    <li><a href="/" class="active">Accueil</a></li>
    <li><a href="/about">À propos</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>
<main>
  <article class="post featured">
    <h2>Premier article</h2>
    <p>Contenu...</p>
  </article>
  <article class="post">
    <h2>Deuxième article</h2>
    <p>Contenu...</p>
  </article>
</main>

Q1 — Tous les liens dans la navigation

.main-nav ul li a

Q2 — Uniquement le lien actif

.main-nav ul li a.active

Q3 — L'article avec la classe featured

article.post.featured

Q4 — Le premier paragraphe de chaque article

article.post p:first-of-type

Q5 — Les éléments <li> pairs

.main-nav ul li:nth-child(even)

1.2 — Spécificité

Q1 — Classement par spécificité croissante

Ordre Sélecteur Spécificité (id, class, tag)
1 (la plus faible)p0, 0, 1
2.text0, 1, 0
3p.text0, 1, 1
4#main1, 0, 0
5 (la plus haute)#main .text p1, 1, 1

Q2 — Si deux règles ont la même spécificité, laquelle gagne ?

La règle déclarée en dernier dans le CSS l'emporte — c'est le principe de cascade (ordre d'apparition).

Q3 — Pourquoi faut-il éviter !important ?

!important court-circuite toute la cascade et la spécificité, ce qui rend le CSS difficile à maintenir et à déboguer. Si plusieurs règles utilisent !important, elles entrent en conflit entre elles et on perd le contrôle de l'ordre de priorité.
